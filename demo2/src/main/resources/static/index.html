Session 事件溯源记忆

JDBC Event Store · RecursiveSummarization 压缩 · conversation_search Recall · SSE 对话

userId = sessionId→ append-only 事件→ Turn 边界压缩→ Recall 检索→ deepseek-chat
✨ 与 ChatMemory 的区别
  • 事件溯源：每条消息为带 UUID 的 SessionEvent，append-only 写入 AI_SESSION_EVENT
  • Turn 安全：压缩以轮次为界，ToolCall + ToolResult 不会被拦腰截断

  • Recall：压缩后可用 conversation_search 按关键词找回早期对话

多 Agent 协作行程规划

Spring AI 实战：Supervisor-Worker 模式 · 1协调+4专项并行+1合并 · DeepSeek 驱动 · 智谱 Embedding 不变

用户需求→ 🧠 Coordinator 拆解→ ⚡并行执行→ 🗺️景点Agent 🍜餐饮Agent 🏨住宿Agent 🚌交通Agent → 🔀 Synthesizer 整合→ ✅完整行程
✨ 多 Agent 协作架构说明
  • CoordinatorAgent（协调者）：提炼需求关键要素（目的地/天数/预算/偏好），供专项 Agent 参考
  • 4 个 Worker Agent 并行执行：AttractionAgent（景点）、DiningAgent（餐饮）、AccommodationAgent（住宿）、TransportAgent（交通）同时调用 DeepSeek
  • SynthesizerAgent（合并器）：整合四个专项结果，生成完整、连贯的最终行程方案
  • 模型配置保持原状：Chat→DeepSeek，Embedding→智谱 embedding-2（本 Tab 无需调用 Embedding）
